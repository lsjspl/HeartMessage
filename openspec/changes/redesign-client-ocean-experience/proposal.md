# 用户端海面体验重设计

## 为什么

当前用户端首页和扔瓶页已经接入真实瓶子接口，但视觉仍是移动端骨架：电脑端留白和比例不合理，海面只是一张小卡片，捡瓶子和扔瓶子缺少动作反馈。Web 是当前第一优先级，H5 是第二优先级，因此需要先把 Web/H5 的核心体验做成可用、好看、可验证的界面。

## 改动内容

- 重做用户端首页为全屏海面场景，兼容桌面端和移动 H5。
- 使用海面、沙滩和瓶子图片生成合成帧动画，呈现动态海浪、漂浮瓶子和捡瓶子过程反馈。
- 重做扔瓶页的书写与封瓶体验，增加瓶子投放动画和额度状态。
- 调整用户端全局样式，让普通页面在桌面端有合理宽度和背景。
- 首页使用轻量岸边操作入口呈现额度和操作，不使用工科化操作面板。
- 不新增依赖，不改变现有瓶子 API 契约。

## 影响范围

- 影响 `apps/client` 的全局样式、首页和扔瓶页。
- 不影响后端接口、数据库、Cloudflare 资源和生产环境。
- 不改变小程序构建链路，样式尽量使用 uni-app 可迁移能力。
