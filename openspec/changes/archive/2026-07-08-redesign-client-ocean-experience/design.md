# 用户端海面体验重设计方案

## 设计目标

用户首次进入首页时应看到明确的“今日海面”产品信号，而不是表单式入口。桌面端要利用宽屏空间呈现沉浸海面和岸边轻量操作入口；移动 H5 要保持单屏核心操作，按钮和额度信息不能挤压或遮挡。

## 首页

首页采用全屏海面场景：

- 背景由用户提供的海面、沙滩和漂流瓶图片生成桌面/移动两套循环帧动画，前端只负责播放、裁切和承接点击，避免运行时用贴纸式素材拼画面。
- 场景需要呈现温暖、有陪伴感的匿名社交氛围：画面重点放在海面、浪花、沙滩和漂流瓶本身，不堆叠热气球、飞机、螃蟹等与核心体验无关的装饰物。
- 中间有漂浮瓶子，空闲时在帧动画中轻微漂浮，捡瓶时页面进入靠岸反馈状态。
- 桌面端呈现完整海面视线，今日额度和操作按钮收在岸边轻量入口中，避免大块操作面板破坏沉浸感。
- 移动端将信息层叠放在海面上方和岸边底部，保证单手可操作。

捡瓶子时先进入 `searching` 状态，接口返回后至少展示一小段动画，再跳转瓶子详情。失败时恢复空闲状态并显示结构化错误消息。

天气联动属于后续数据能力：页面样式需要预留晴天、阴天、雨天和风天的场景 class，但真实天气接口、定位授权、城市选择、缓存和失败态必须另开 OpenSpec change 后再实现。

## 扔瓶页

扔瓶页保留真实提交逻辑，但视觉从普通表单改为“写信、封瓶、推向海面”：

- 左侧或上方是信纸式文本输入区。
- 右侧或下方显示瓶子预览、匿名开关和今日剩余额度。
- 提交时进入 `launching` 状态，瓶子向海面移动，成功后再跳转详情页。

## 样式边界

本次只改用户端样式和页面结构，不新增 UI 依赖，不修改共享 DTO，不改后端服务。全局样式只提供页面背景、按钮、输入框和桌面最大宽度，不承载业务逻辑。
