# 用户默认头像库

## Purpose

资料完善页目前只能上传头像。Web 用户在首次登录时不一定愿意立刻上传本地照片，因此需要提供一组可直接选择的默认头像，让用户不用上传也能完成资料设置。头像风格需要与 HeartMessage 的海面、漂流瓶和温柔陪伴气质一致。

## What Changes

- 新增内置默认头像资产：男 4 个、女 4 个、动物动漫 3 个。
- 用户端资料页展示默认头像选择网格，并允许与上传头像共存。
- 用户选择默认头像后，保存资料时直接提交对应静态资源 URL。
- 上传头像仍作为可选入口保留，上传成功后可覆盖当前默认头像选择。

## Impact

- 影响 `apps/client` 资料完善页和静态资源。
- 不影响 API、数据库和现有资料保存契约。
- 不新增依赖，不执行生产环境操作。
