# 瓶子与聊天闭环

## 为什么

漂流瓶产品的核心体验是扔瓶子、捡瓶子、回复并开始聊天。当前项目已经有数据库表和页面骨架，但 API 仍返回 demo 数据，用户无法真正消耗每日额度、捞取真实瓶子、回复瓶子或删除会话关系。需要先把 Web/H5 的核心闭环做成可验证的真实功能。

## 改动内容

- 实现每日额度查询，用户每天最多捡 20 个瓶子、扔 3 个瓶子。
- 实现扔瓶子接口，瓶子当天有效，过期后不再被捞取。
- 实现捡瓶子接口，只捞取未过期、未被捡走、非本人扔出的漂流瓶。
- 实现回复瓶子接口，回复后创建会话和首条消息。
- 实现聊天列表、消息列表、发送消息和删除会话关系接口。
- 用户端页面接入真实 API，去掉固定 demo 流程。
- 管理后台增加瓶子列表基础数据来源。

## 影响范围

- 影响 `apps/api` 的瓶子、聊天和后台路由。
- 影响 `apps/client` 的首页、扔瓶子、瓶子详情、聊天列表和聊天详情。
- 影响 `apps/admin` 的瓶子管理基础页。
- 影响 `packages/shared` 的瓶子、聊天 DTO。
- 可能需要补充 D1 migration，用于支持会话可见性和消息读取状态。
