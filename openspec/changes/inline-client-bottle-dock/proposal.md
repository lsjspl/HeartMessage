# 首页瓶子弹窗收件箱

## 为什么

当前用户端首页捡到瓶子后会跳转到瓶子详情页，AI 补位生成的瓶子也沿用跳转流程，用户会感觉内容没有在当前海面弹出来。聊天列表和聊天详情已经存在，但需要从首页右上角的瓶子入口进入一个统一弹窗，像聊天软件一样左侧浏览瓶子列表、右侧查看瓶子内容或继续聊天，减少多页面跳转。

## 改动内容

- 首页捡瓶成功后在当前页面打开瓶子弹窗，并展示刚捡到的瓶子内容。
- 首页右上角展示用户瓶子数量入口，点击打开同一个弹窗。
- 弹窗左侧展示用户捡到和扔出的瓶子列表，右侧展示瓶子内容、回复入口或聊天窗口。
- 新增只读的用户瓶子列表 API，复用现有瓶子详情、回复和聊天消息 API。
- 不改变现有捡瓶、扔瓶、回复和聊天 API 的语义，不新增依赖。

## 影响范围

- 影响 `packages/shared` 的用户瓶子列表 DTO。
- 影响 `apps/api` 的瓶子路由和服务。
- 影响 `apps/client` 首页、瓶子服务和首页组件。
- 不涉及数据库结构、生产环境配置和 Cloudflare 资源。
