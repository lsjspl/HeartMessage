# 后台导航与列表拆分

## 为什么要做

当前后台存在复杂页面平铺多个列表和配置区的情况，特别是 AI 模型管理把供应商、模型和用途绑定放在同一页面。随着内容安全和用户画像加入，继续平铺会让后台难以扫描和维护。

## 要改什么

- 后台侧边栏改为按模块分组。
- 复杂模块使用子菜单。
- 一个管理页面只承载一个主要列表。
- AI 模块拆分为供应商、模型、用途绑定三个页面。
- 用户模块增加用户画像子页面。
- 工作台只保留指标概览，不再平铺多个管理列表。

## 影响范围

- `apps/admin` 路由、侧边栏和相关页面。
- 后台 API 仅按新增画像页面补充接口，不改变已有接口路径。

## 非目标

- 不更换 UI 框架。
- 不重写登录鉴权。
- 不改变已有用户、瓶子、日志接口语义。
