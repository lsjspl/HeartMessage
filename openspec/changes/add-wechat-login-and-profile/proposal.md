# 微信登录与资料完善

## 为什么

瓶子、聊天、每日额度、删除关系和后台用户管理都依赖稳定的用户身份。当前项目只有 demo 登录返回值，需要先把微信登录注册、Token 会话、个人资料完善和头像上传入口打通，后续功能才能基于真实用户推进。

## 改动内容

- 增加微信网页授权登录接口，支持本地开发占位登录。
- 增加当前用户查询接口。
- 增加个人资料创建/更新接口。
- 增加头像上传申请接口，为 R2 上传预留对象 key。
- H5 用户端接入登录、资料保存和本地 token 持久化。
- 管理后台增加用户列表基础页。

## 影响范围

- 影响 `apps/api` 的认证中间件、用户接口和 Cloudflare 环境变量。
- 影响 `apps/client` 的登录页、资料页、session store 和 API 客户端。
- 影响 `apps/admin` 的用户管理页。
- 影响 `packages/shared` 的认证和资料相关 DTO。
