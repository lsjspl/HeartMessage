# 后台管理与 AI 模型配置

## 为什么

管理后台目前可以打开页面，但缺少管理员登录、真实参数保存、AI 模型维护、用途绑定和日志查询。漂流瓶产品后续需要通过后台控制每日额度、AI 补位策略、AI 供应商模型、用户瓶子数据和操作审计，因此必须先把后台管理闭环做成真实能力。

## 改动内容

- 增加管理员账号密码登录，返回仅用于后台的 admin token。
- 除 `/admin/auth/login` 外，所有 `/admin/*` 接口必须要求管理员鉴权。
- 增加系统参数读取和保存，参数存入 Cloudflare KV。
- 增加 AI 供应商和模型管理接口，模型配置存入 D1。
- 增加 AI 用途绑定配置，用于选择人格生成、瓶子生成、聊天回复和内容审核模型。
- 增加操作日志写入和查询。
- 管理后台页面接入真实登录、参数、AI 模型和日志 API。

## 影响范围

- 影响 `apps/api` 的 admin、ai、日志、鉴权和 Cloudflare 环境变量。
- 影响 `apps/admin` 的登录流程、路由守卫、系统参数页、AI 模型页和日志页。
- 影响 `packages/shared` 的后台 DTO。
- 影响 `packages/db` 和 D1 migration，用于 AI 模型唯一约束或日志字段扩展。
