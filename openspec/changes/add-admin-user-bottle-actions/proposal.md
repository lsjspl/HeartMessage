# 后台用户与瓶子管理操作

## Purpose

当前管理后台已经有用户列表和瓶子列表，但仍缺少实际管理动作。为了满足 MVP 中“用户管理功能、瓶子管理功能”的要求，需要让管理员可以禁用或恢复用户账号，并能对瓶子执行封禁、删除、过期等处理，同时写入操作日志。

## What Changes

- 增加后台用户列表共享 DTO。
- 增加用户状态更新接口，支持 `active` 和 `disabled`。
- 被禁用的普通用户 token 再访问用户端接口时 MUST 被拒绝。
- 增加瓶子状态更新接口，支持后台将瓶子设为 `blocked`、`deleted` 或 `expired`。
- 瓶子被封禁或删除时，关联会话不再保持 active。
- 管理后台用户页和瓶子页接入操作按钮。

## Impact

- 影响 `packages/shared`。
- 影响 API 后台路由、鉴权中间件和瓶子服务。
- 影响管理后台用户管理页和瓶子管理页。
- 不涉及数据库结构变更。
