# 头像 R2 上传与资料页完善

## Purpose

用户资料要求包含头像、昵称、个人介绍、年龄和性别。当前系统已有头像上传申请接口，但还没有真正接收文件写入 R2，也没有公开读取头像的媒体路由；用户端资料页也没有接入选择头像和性别。需要补齐这条链路，让 Web/H5 用户可以选择头像、上传到 Cloudflare R2，并保存到个人资料。

## What Changes

- 完善头像上传接口：申请受控对象 key 后，用户可以把图片上传到 API，API 写入 R2。
- 增加媒体读取路由，用于读取 R2 中的头像对象。
- 用户端资料页接入 `chooseImage`、上传进度、头像预览和性别选择。
- 保存资料时提交头像 URL 和性别。
- 上传失败必须显式报错，不得保存未上传成功的头像地址。

## Impact

- 影响 `apps/api` 上传和媒体路由。
- 影响 `apps/client` 资料完善页和上传 service。
- 复用现有 shared DTO，不新增依赖。
- 不执行生产 R2 操作，只在本地 Worker/R2 绑定中验证。
