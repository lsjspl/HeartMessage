# 后台拉取供应商模型列表设计

## API 设计

后台新增只读接口：

- `GET /admin/ai/providers/:providerId/models`

接口读取 `ai_providers` 中的供应商配置，使用该供应商的 `adapter_type` 选择适配器，并通过 `api_key_secret_name` 从后台敏感配置读取 API Key。接口只返回模型标识、展示名和归属摘要，不返回密钥。

## 适配器设计

`packages/ai` 的供应商适配器增加 `listModels` 能力。首个实现仍为 `openai_compatible`，请求 `{baseUrl}/models`，解析 OpenAI-compatible 响应中的 `data[].id`。

如果供应商未配置 `baseUrl`、API Key 缺失、适配器不支持拉取、或供应商接口失败，API MUST 显式返回结构化错误，不得静默返回空列表。

## 后台交互

模型表单的“模型名”控件改为可搜索、可输入的选择框。管理员选择供应商后点击拉取按钮，后台请求接口并把模型列表填入下拉选项。

展示名默认规则：

- 新增模型时，展示名为空。
- 当管理员选择或输入模型名，且展示名未被手动修改时，自动填充为“用途名称 + 模型名”。
- 当用途变化且展示名仍为自动生成值时，同步更新展示名。
- 编辑已有模型时保留已有展示名，除非它本来就是默认展示名。
