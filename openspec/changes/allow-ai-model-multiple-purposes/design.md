# AI 模型支持多用途设计

## 数据结构

`ai_models` 使用 `purposes_json` 字段保存模型支持的用途数组。迁移完成后删除旧 `purpose` 字段，不保留单用途旧结构。

迁移会把已有 `purpose` 写入 `purposes_json`。历史上如果同一供应商同一模型名已经有多行，迁移会合并为一条，更新用途绑定和模型引用，删除重复行。

## API

共享契约：

- `AiModelUpsertInput.purposes`：至少一个用途。
- `AdminAiModel.purposes`：模型支持的用途数组。

保存模型时，如果新增记录与已有记录拥有相同 `providerId + modelName`，API MUST 返回冲突错误。编辑时如果改成另一个已存在模型名，也 MUST 返回冲突错误。

删除模型时，API 删除模型记录，并清理系统设置中指向该模型的用途绑定。

## 后台

模型表单的用途类型使用多选。展示名默认使用“用途组合 + 模型名”，例如“聊天回复/画像评估 deepseek-chat”。模型列表用途列展示多个用途标签。

用途绑定页面只展示包含当前用途且启用的模型。

## 运行时

AI 运行时仍按系统设置中的用途绑定读取模型 ID。读取模型时必须确认该模型的 `purposes_json` 包含当前用途，模型或供应商停用时继续显式报错。
