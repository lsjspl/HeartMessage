# AI 模型支持多用途

## Purpose

后台新增模型时，一个供应商下同一个模型名不应该因为用途不同被添加成多条记录。管理员需要一次选择多个用途，列表也按“供应商 + 模型名”展示为一行，避免模型配置散乱和重复。

## Scope

- 模型保存契约从单用途改为多用途。
- 后台模型表单用途改为多选。
- 后台模型列表按供应商和模型名展示为一行。
- API 保存模型时拒绝同供应商同模型重复新增。
- AI 运行时按用途判断模型是否可用。
- 数据库新增用途数组字段，并迁移旧单用途数据。

## Non-Goals

- 不改变用途绑定仍然按用途选择模型 ID 的方式。
- 不执行生产数据库迁移。
